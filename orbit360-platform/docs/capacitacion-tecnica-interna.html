🏗 Arquitectura del sistema

🧑‍💻
Frontend (Hoy)

HTML + JS vanilla (Orbit.modules), CSS tokens, sin frameworks. 100% en el navegador con localStorage como capa de datos.

🗄
Backend (Migración)

Reemplazar Orbit.store por Firebase/Supabase/Postgres. Los módulos solo llaman a Orbit.store — no tocar nada más.

🏢
Multi-tenant

Orbit.tenant por cliente (branding, plan, módulos activos, roles). La personalización de A&S es solo config — no modificar el core.

⚡
Automatizaciones

Orbit.modules.automatizaciones.disparar(evId, datos) → Make webhook → WA/correo/Sheets. Sin Make: solo se loguea.

5
Verificar y lanzar
Revisar 5 clientes de prueba end-to-end (crear, póliza, cobro, renovación). Cuando todo cuadre, compartir la URL con el equipo.

🔌 Conectar integraciones reales

Gemini IA (importador inteligente real)

1. Ir a aistudio.google.com → Get API key → Copiar 2. Orbit 360 → Automatizaciones → Asistente IA → Pegar API key → Activar → Guardar 3. El importador ya usará extracción real de PDFs en lugar de heurística.

Make (automatizaciones)

1. make.com → Crear escenario → Webhook → Copiar URL 2. Orbit 360 → Automatizaciones → Webhook Make → Pegar URL → Guardar 3. Probar disparo → verificar en Make que llega el payload 4. Desde Make: ramificar a WhatsApp Cloud / Gmail / Google Sheets

Outlook / Microsoft 365

1. portal.azure.com → Registrar app → Permisos: Mail.ReadWrite, Mail.Send, Calendars.Read 2. Copiar client_id + client_secret + tenant_id 3. Orbit 360 → Configuración → Integraciones → Outlook → Configurar (En producción, agregar estos parámetros al config del tenant)

🩺 Soporte y troubleshooting

  • La app carga en blanco: abre DevTools (F12) → Console → buscar el error. Más del 80% son problemas del seed (__v desactualizado) o un import faltante en index.html.
  • Los datos no aparecen: verifica que seed.__v cambió correctamente. Si el store tiene una versión anterior, resetea con Orbit.store.reseed(Orbit.SEED) en la consola.
  • Un módulo no carga: asegúrate de que el script esté en index.html DESPUÉS de sus dependencias (config.js → primas.js → ciclo.js → módulos).
  • El importador no abre el selector: debe haber un fileInput dentro del drawer con id="imp-file". Verifica que el click listener esté en el input, no solo en el drop.
  • Error 153 en videos YouTube: el video no permite embedding. Usa videos propios en Drive/Vimeo o el placeholder nativo de Orbit Academia.

📋 Cómo agregar un nuevo cliente (tenant)

// 1. Copia la carpeta orbit360-platform/ con un nuevo nombre // 2. Edita core/config.js: Orbit.tenant = (function () { const data = { nombre: 'Mi Intermediaria S.A.', modulos: [...todos los del plan...], branding: { logo: '', paleta: 'rojo', tipografia: 'Manrope' } }; ... }); // 3. Personaliza seed.js: aseguradoras locales, metas, asesores // 4. Sube a GitHub Pages o hosting → URL lista
💡 Regla de oro: NUNCA modifiques el core para personalizar un cliente. Toda la personalización va en config.js y seed.js. Así puedes actualizar el core de todos los tenants sin romper nada.
