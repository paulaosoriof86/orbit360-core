❓ Preguntas clave antes de empezar

¿Proyecto nuevo o continúo aquí?

Proyecto nuevo en Claude. Este proyecto ya acumula 580+ turnos de construcción. Para la migración A&S conviene una pizarra limpia que solo contenga el código, no el historial de build.

¿Claude o ChatGPT?

Claude (Anthropic) obligatoriamente. El código de Orbit 360 vive en proyectos de Claude; solo Claude puede leer, editar y desplegar los archivos nativamente. ChatGPT no tiene acceso al filesystem del proyecto.

¿Proyecto normal o especial?

Proyecto normal de Claude Pro/Team. No necesitas una cuenta técnica especial — el mismo plan que usas aquí.

¿Qué sube al nuevo proyecto?

  • El ZIP más reciente de este proyecto (orbit360-platform/) — es el estado completo.
  • Un CLAUDE.md nuevo (ver más abajo) que le da todo el contexto sin los 580 turnos.
  • Los documentos de A&S que necesites migrar (pólizas, clientes, estados de cuenta).

¿Sigo en el proyecto de la plataforma antigua de A&S?

No. La plataforma antigua de A&S (cotizador/comparativo legacy) es una herramienta separada. El proyecto nuevo de migración es independiente y solo usa Orbit 360 como base.

📍 Pasos de migración (en orden)

1
Branding A&S
Subir logo en Configuración → Marca. Cambiar nombre del producto. Ajustar paleta de colores si difiere del rojo Orbit. El sidebar y el login cambian automáticamente.
2
Configurar aseguradoras reales de GT
En Orbit Aseguradoras: habilitar las vinculadas a A&S (El Roble, G&T, Mapfre…); cargar NIT, patrón de concepto, cuentas bancarias, ejecutivos de cuenta y documentos de cada una. El importador documental puede extraer NIT/patrón de facturas previas.
3
Migrar clientes y pólizas
Desde Importación Inteligente → subir los archivos del histórico de A&S (Excel/CSV/PDF). El motor detecta clientes, pólizas y vehículos, cruza por identificación/N.º de póliza y complementa sin duplicar. Señala lo que no puede leer para completar manual.
4
Conectar integraciones
En Configuración → Integraciones: (1) Outlook/Gmail para la bandeja interna, (2) Google Drive para los expedientes, (3) WhatsApp Business API para cadencias y recordatorios, (4) Make para automatizaciones de cobro/renovación, (5) Gemini API key para importador inteligente real.
5
Tarifas de comisiones
En Comisiones → Tarifas o Equipo → Comisiones: configurar % por ramo por aseguradora e importar la planilla de comisiones mensual. La plataforma calcula automáticamente comisión aseguradora y vendedor.
6
Usuarios y roles del equipo A&S
En Equipo y Permisos: crear un usuario por asesor con su rol (Asesor/Admin/Dirección/Marketing), conectar su correo de plataforma y asignar módulos visibles. Al crear, se envía bienvenida + credenciales por correo/WA.
7
Verificar y poner en producción
Subir el código a GitHub → activar GitHub Pages (o hosting propio). URL tipo: https://tuusuario.github.io/orbit360-as/ . Compartir el link con el equipo. El Portal del Cliente tiene su URL separada.

🔗 Integraciones que requieren credenciales reales (en migración)

✉
Correo (Outlook/Gmail)

Registrar app en Azure AD (Outlook) o Google Cloud Console (Gmail) → OAuth2 → pegar client_id + secret en Config → Integraciones.

💬
WhatsApp Business API

Meta Business → WhatsApp API → token permanente → pegar en Config. O usar Green API como proxy.

📁
Google Drive

Google Cloud Console → Service Account → habilitar Drive API → JSON de credenciales → Config → Integraciones → Drive.

✨
Gemini (IA real)

Google AI Studio → Crear API key → pegar en Automatizaciones → Asistente IA → Guardar. Activa el importador inteligente real y el análisis crítico.

🔗
Make (automatizaciones)

Make.com → Crear escenario → Copiar webhook URL → pegar en Automatizaciones → Webhook de Make → Guardar. Cada evento de Orbit disparará el escenario.

🗄
Backend / Base de datos

Para producción real: reemplazar Orbit.store por Firebase/Supabase. El código está diseñado para este swap — los módulos solo llaman a Orbit.store.all/get/insert/update.
